# Testing Guide\n\n## Enterprise Test Suite\n\n### Test Structure\n```\ntests/\n├── booking/                  # Booking Domain Tests (21 tests)\n│   ├── test_booking_repository.py\n│   ├── test_booking_service.py\n│   └── test_booking_controller.py\n├── checkin/                  # Check-in Domain Tests (10 tests)\n│   └── test_checkin_service.py\n├── flight/                   # Flight Domain Tests (9 tests)\n│   └── test_flight_passenger_repositories.py\n└── shared/                   # Shared Component Tests (17 tests)\n    └── test_exceptions.py\n```\n\n### Running Tests\n\n#### All Enterprise Tests\n```bash\ncd backend\npython tests/test_enterprise_coverage.py\n```\n\n#### Domain-Specific Tests\n```bash\n# Booking domain\npytest tests/booking/ --cov=app.booking -v\n\n# Check-in domain\npytest tests/checkin/ --cov=app.checkin -v\n\n# All tests with coverage\npytest tests/ --cov=app --cov-report=html --cov-report=term-missing\n```\n\n### Coverage Requirements\n- **Minimum**: 95% code coverage\n- **Total Test Cases**: 57 comprehensive tests\n- **Coverage Target**: All enterprise modules\n\n### Test Categories\n\n#### Repository Tests\n- Database operations\n- Error handling\n- Edge cases\n- Data validation\n\n#### Service Tests\n- Business logic validation\n- Transaction management\n- Exception scenarios\n- Integration testing\n\n#### Controller Tests\n- HTTP endpoint testing\n- Authentication validation\n- Request/response handling\n- Error response testing\n\n#### Exception Tests\n- All custom business exceptions\n- Error message validation\n- Status code verification\n- Exception inheritance\n\n### Test Quality Standards\n\n#### Mocking Strategy\n- Use `AsyncMock` for async operations\n- Mock external dependencies\n- Isolate units under test\n\n#### Test Scenarios\n- **Success Cases**: Happy path testing\n- **Failure Cases**: Error condition testing\n- **Edge Cases**: Boundary condition testing\n- **Integration**: Cross-domain testing\n\n#### Assertions\n- Verify return values\n- Check method calls\n- Validate state changes\n- Confirm error handling\n\n### Coverage Reports\n\n#### HTML Reports\n```bash\npytest --cov=app --cov-report=html\n# Open htmlcov/index.html\n```\n\n#### Terminal Reports\n```bash\npytest --cov=app --cov-report=term-missing\n```\n\n#### Coverage Thresholds\n```bash\npytest --cov=app --cov-fail-under=95\n```\n\n### Continuous Integration\n\nTests run automatically on:\n- Pull requests\n- Main branch commits\n- Release tags\n\nCoverage requirements must be met for CI to pass.
