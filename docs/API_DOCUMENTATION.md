# API Documentation\n\n## Enterprise API Endpoints\n\n### Authentication\n- `POST /auth/register` - User registration\n- `POST /auth/token` - JWT token generation\n\n### Booking Domain\n- `POST /api/bookings` - Create new booking\n- `GET /api/bookings/{id}` - Get booking details\n- `DELETE /api/bookings/{id}` - Cancel booking\n\n### Check-in Domain\n- `POST /api/checkin` - Process web check-in\n- `GET /api/checkin/{id}` - Get boarding pass\n- `GET /api/checkin/status/{booking_id}` - Check status\n\n### Security\nAll API endpoints require JWT authentication except:\n- `/auth/register`\n- `/auth/token`\n- `/health`\n\n### Error Responses\nStandardized error format:\n```json\n{\n  \"error\": true,\n  \"message\": \"Error description\",\n  \"type\": \"ExceptionType\",\n  \"path\": \"/api/endpoint\"\n}\n```\n\n### Status Codes\n- `200` - Success\n- `201` - Created\n- `204` - No Content\n- `400` - Bad Request\n- `401` - Unauthorized\n- `404` - Not Found\n- `409` - Conflict\n- `422` - Validation Error\n- `500` - Internal Server Error
